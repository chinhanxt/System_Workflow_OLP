Trunk-Based Development

Workflow gộp code siêu ngắn - Động cơ thúc đẩy CI/CD

  • Đồng bộ mã nguồn mới nhất từ main:

    Luôn cập nhật để tránh xung đột code.

  • Tạo nhánh tính năng siêu ngắn hạn (sống dưới 24h):

    Phục vụ các commit cực nhỏ.

  • Commit và push liên tục trong ngày:

    Tích hợp code ngay khi hoàn thành một phần nhỏ.

  • Gộp trực tiếp vào main sau khi Test pass tự động:

    Sử dụng Feature Flags để bật/tắt tính năng.

Trunk-Based Script
$git checkout main
$git pull origin main
$git checkout -b feat/small-toggle
# Code và commit ngay lập tức
$git add .
$git commit -m "feat: add feature flag"
$git push origin feat/small-toggle

GitHub Flow

Mô hình Nhánh Tính Năng (Feature Branch) cho Web & SaaS

  • Tạo nhánh tính năng có tên rõ ràng:

    Ví dụ: feature/login-page.

  • Phát triển tính năng và tạo commit cục bộ:

    Mỗi commit đại diện cho một phần hoàn thiện.

  • Đẩy nhánh lên GitHub & mở Pull Request (PR):

    Yêu cầu đội ngũ Review Code.

  • Thảo luận, chỉnh sửa code và Merge vào main:

    Deploy trực tiếp từ main lên môi trường Production.

GitHub Flow Script
$git checkout -b feature/login-page
$git add .
$git commit -m "feat(auth): add UI login"
$git push -u origin feature/login-page
# Gộp sau khi PR được duyệt
$git checkout main && git merge feature/login-page

Git Flow

Phát hành (Release) chặt chẽ cho Mobile App và Thư viện SDK

  • Feature phát triển từ develop:

    Làm việc trên nhánh feature/*, merge về develop.

  • Tạo nhánh Release để chuẩn bị phát hành phiên bản:

    Nhánh release/vX.Y.Z chỉ để fix bug.

  • Merge Release vào cả main và develop:

    Gắn nhãn (Tag) phiên bản trên nhánh main.

  • Hotfix khẩn cấp trực tiếp từ main:

    Tạo nhánh hotfix/* sửa lỗi gấp và merge lại.

Git Flow Script
$git checkout -b release/v1.0.0 develop
# Sửa lỗi phát hành...
$git checkout main && git merge release/v1.0.0
$git tag -a v1.0.0 -m "Tag release"
$git checkout develop && git merge release/v1.0.0

GitLab Flow

Liên kết nhánh Git trực tiếp với các Môi trường triển khai (Environments)

  • Mọi tính năng được tích hợp trước vào main:

    Main là trung tâm tích hợp liên tục.

  • Chuyển mã nguồn lên môi trường Staging (Kiểm thử):

    Merge nhánh main vào nhánh staging.

  • Phát hành lên Production (Môi trường thật):

    Merge nhánh staging vào nhánh production.

  • Sửa lỗi xuôi dòng (Cherry-pick):

    Fix bug ở main trước, sau đó gộp vào các nhánh staging/production.

GitLab Flow Script
# Tải lên môi trường Staging
$git checkout staging && git merge main
$git push origin staging
# Tải lên Production chính thức
$git checkout production && git merge staging
$git push origin production

Cheat Sheet: Lệnh Xử Lý Sự Cố Khẩn Cấp

Các câu lệnh giải cứu khi gõ nhầm hoặc xảy ra xung đột code (Conflict)

1. HUỶ COMMIT HOẶC KHÔI PHỤC FILE
# Huỷ commit gần nhất nhưng giữ lại code đã viết
$git reset HEAD~1
# Huỷ bỏ toàn bộ code đã sửa chưa commit
$git reset --hard HEAD
# Khôi phục 1 file cụ thể về phiên bản commit trước
$git checkout HEAD -- filepath
2. LƯU TẠM & GIẢI QUYẾT XUNG ĐỘT
# Cất giữ tạm thời code đang viết để pull code mới
$git stash
# Lấy lại code đã cất giữ sau khi đã pull
$git stash pop
# Xem các tệp tin đang bị xung đột (Conflict)
$git diff --name-only --diff-filter=U
