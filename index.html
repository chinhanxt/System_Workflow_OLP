WORKFLOW 1
Trunk-Based Development

Chu kỳ gộp code siêu ngắn - Động cơ thúc đẩy CI/CD

Quy trình hoạt động từng bước của Trunk-Based:

  1. Lấy code mới nhất: Luôn cập nhật nhánh chính trước khi code.
  2. Tạo nhánh ngắn hạn: Nhánh chỉ tồn tại dưới 24h.
  3. Tạo Pull Request nhỏ: Code ít, review nhanh, merge ngay lập tức.
  4. Tự động hóa hoàn toàn: CI chạy test tự động khi merge vào main.
bash - trunk_dev.sh
$git checkout main
$git pull origin main
$git checkout -b feat/small-fix
# Viết code và commit nhanh
$git add . && git commit -m "feat: add flag"
$git push origin feat/small-fix

WORKFLOW 2
GitHub Flow

Chuẩn mực và phổ biến nhất cho sản phẩm Web & SaaS

Quy trình 6 bước chuẩn chỉnh:
1

Create Branch

Tạo nhánh từ main với tên có ý nghĩa rõ ràng.

git checkout -b feature/login
2

Add Commits

Lưu trữ các thay đổi cục bộ kèm thông điệp chuẩn.

git commit -m "feat: UI login"
3

Open PR

Tải lên remote và mở Pull Request để xin review.

git push -u origin feature/login
4

Merge & Deploy

Merge vào main và deploy tự động lên web.

git merge feature/login

WORKFLOW 3
Git Flow

Mô hình chặt chẽ cho sản phẩm Release định kỳ (Mobile App/SDK)

Cơ cấu các nhánh trong Git Flow:

  • main: Chỉ chứa code đã release chạy ổn định.
  • develop: Nhánh phát triển chính, gộp các feature.
  • feature/*: Nhánh tính năng con, làm xong merge về develop.
  • release/*: Chuẩn bị phát hành, chỉ fix bug tại đây.
  • hotfix/*: Vá lỗi khẩn cấp trực tiếp từ main.
bash - release_flow.sh
$git checkout develop
$git checkout -b release/v1.0.0
# Fix bugs...
$git checkout main
$git merge release/v1.0.0
$git tag -a v1.0.0 -m "Release v1.0.0"

WORKFLOW 4
GitLab Flow

Tích hợp chặt chẽ giữa nhánh Git và Môi trường chạy thực tế (Production)

Quy trình định hướng môi trường:

  1. Tính năng mới: Phát triển trên các nhánh feature và gộp vào main.
  2. Staging (Kiểm thử): Merge code từ main vào staging để kiểm thử.
  3. Production (Vận hành): Gộp code từ staging vào nhánh production khi đã an toàn.
bash - deploy_env.sh
# Deploy lên Staging
$git checkout staging && git merge main
$git push origin staging
# Promotion lên Production
$git checkout production && git merge staging
$git push origin production

Quy Tắc Vàng Khi Làm Việc Team

Đặt tên nhánh & Commit chuẩn quốc tế

Đặt tên nhánh (Branch Name)

  • feature/task-id-mo-ta: Tính năng mới.
  • bugfix/task-id-mo-ta: Sửa lỗi trong sprint.
  • hotfix/task-id-mo-ta: Vá lỗi khẩn cấp trên Production.

Thông điệp Commit (Conventional Commits)

  • feat(auth): add google sign-in
  • fix(cart): resolve quantity overflow
  • refactor(db): optimize query user profiles
  • chore(deps): update lodash library
